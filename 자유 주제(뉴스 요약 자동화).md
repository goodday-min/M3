# [프로젝트 2] 뉴스 요약 자동화

-------

매일 수많은 뉴스가 쌓이는 정보중에서 "내가 정말 놓치면 안 되는 중요한 키워드"나 "특정 주제"가 포함된 정보들을 요약된 내용을 모아서 볼 수 있는 시나리오입니다.<br>

목표<br>
*RSS 피드에서 자동으로 뉴스를 수집하고, AI로 요약한 후, 노션 데이터베이스에 저장하는 완전 자동화 워크플로우 구축*

-------

1. 구조 요약:
    - Trigger (1개): RSS (새 정보 도착)
    - Filter (1개): 필터를 사용하여 뉴스 제목에 '특정 키워드(네이버)'가 있으면 넘어 감
    - Action (1개): 노션 기록


2. 주요 기능
✅ RSS 피드 자동 감지 (새 항목만)
✅ 키워드 필터링 (네이버 주식 관련)
✅ AI 요약 (OpenAI GPT-4o-mini)
✅ URL 자동 정제 및 디코딩
✅ 노션 자동 저장

3. 사용 도구
| 도구 | 역할 | 
| Make.com | 	워크플로우 자동화 플랫폼 | 
| Google Alerts | RSS	뉴스 피드 소스 | 
| OpenAI API	| AI 요약 및 키워드 추출 | 
| Notion API | 	데이터베이스 저장 | 

2. 워크플로우 흐름:
    - [Trigger 1] RSS에 새 뉴사가 도착합니다.<br>
    - [공통 Action 1] 우선 모든 정보는 기존처럼 구글 스프레드시트에 차곡차곡 기록합니다.<br>
    - [조건 분기: Router] 구글 시트 모듈 뒤에 Router를 연결하여 길을 2개로 나눕니다.<br>
       상단 길 (Filter): 제목에 '긴급' 또는 '필독' 혹은 내가 설정한 '핵심 키워드'가 포함되어 있는 경우만 통과.<br>
       하단 길 (Filter): 그 외 일반적인 정보인 경우 통과 (혹은 아무것도 하지 않음).<br>
    - [Action 2 - 상단 길] 중요 키워드가 걸렸다면, 내가 자주 쓰는 슬랙(Slack) 이나 텔레그램, 메신저 또는 메일로 "🚨 중요 정보가 수집되었습니다! [링크]" 하고 즉시 팝업 알림을 보냅니다.<br>
    - [Action 3 - 하단 길] 일반 정보라면, 일주일에 한 번 모아서 볼 수 있도록 다른 노션(Notion) 데이터베이스로 이동시키거나 그냥 시트 기록으로 마무리합니다.<br>


3. 자동화 도구 : MAKE 선정
     - 선정이유 : <br>
       1. 하나의 화면에서 여러 개의 분기(Router)와 반복문(Iterator/Aggregator)을 한눈에 볼 수 있어 **시각적인 데이터 흐름 및 분기 파악 용이**
       2. 기본 제공하는 앱의 절대적인 수는 Zapier보다 적지만 API 통신 기능이 매우 강력하여 **기술적 확장성**이 뛰어남
       3. **비용 효율성**이 매우 뛰어남 (동일 작업량 대비 Zapier 비용의 수분의 일 수준)
       4. **무료 플랜**에서도 조건부 분기를 지원하여 소규모 비즈니스나 테스트용으로 매우 훌륭함
